用 hugo 搭建博客

2020-08-03
随笔

写在前面：这篇文章旨在记录一个普通的非计算机专业学生在 Windows 10 系统里折腾 Hugo 博客的过程，作备忘之用。如果有相似处境的人读到了这篇文档，并从中得到了一些建设 Hugo 博客的灵感，作者不胜荣幸。

1） 准备工作

准备好一台 Windows 电脑。这台电脑里需要有一个 Markdown 编辑器，系统自带的 Notepad 就可以。我使用的编辑器是 Typora。

如果你不知道自己的电脑是 32 位还是 64 位，进入控制面板——系统和安全——系统，可以在这里查看。

在 D 盘里创建一个文件夹 D:\hugo ，在这个文件夹内再创建两个文件夹：D:\hugo\bin 和 D:\hugo\sites。当然你也可以建在别的位置，但我是这样建的。

2） 安装 Hugo

前往 https://github.com/gohugoio/hugo/releases 安装电脑相应版本的 Hugo。下载系统对应版本的 zip 文件即可，不要下载源代码不过大佬做什么都可以。你会得到一个压缩文件。

将压缩文件放到D:\hugo\bin中，解压到该文件夹。你会得到三个文件，其中有一个是可执行文件（.exe）。

下面在 windows 电脑中搜索”环境变量“。窗口的上半部分有一个”用户的环境变量“栏。在此栏中找到 Path 一项，向其中添加一条路径 D:\hugo\bin。

Win 7 和 Win 8 不支持自定义路径。如果使用这两个操作系统，建议下载一个第三方编辑器。目前这个网址可以在国内以正常速度打开。

按 Windows 键 + R，在出现的窗口中键入 cmd（或 powershell），然后回车，会出现一个深色的命令行窗口。在这个窗口中键入 hugo version，如果出现类似下图的结果：

1

Hugo Static Site Generator v0.74.3-DA0437B4 windows/amd64 BuildDate: 2020-07-23T16:23:30Z

则说明你已经安装成功了。

如果安装不成功，请检查一下是否下载了正确版本的 Hugo，环境变量是否配置正确。

3） 创建一个新的网站

在命令行窗口中进入 hugo sites 文件夹（如果不会的话，参看这里），输入 hugo new site [example]，即可创建一个新的网站。 把 [example] 替换成你想要的站点名称。

完成后，会在 D:\hugo\sites下生成一个新的文件夹，形成：D:\hugo\sites\[example]。在这一文件夹下，会有数个子文件夹和一个 config.toml 文件。

4） 加载主题

作者私心：这一步我卡了好久好久好久。

首先，进入 https://themes.gohugo.io/ 挑一个你喜欢的主题。点击下载按钮后，会跳转到一个 github 页面。点击页面中绿色的 Code 按钮，在下拉框中选中 Download ZIP，即可下载主题文件夹。作者使用的主题是 even，下文会以此为例。

将主题文件夹解压后重命名为 even。

下面，进入 D:\hugo\sites\[example]\themes\tokiwa\exampleSite\config.toml，将 config.html 打开，把其中的内容复制到 [example] 文件夹中的 config.toml 文件里。然后打开 [example]\config.html，将其中的 theme 一项改为：

1

theme = "even"

再添加一行代码：

1

themeDir = "theme"

然后在命令行界面内进入 D:\hugo\sites\[example] ，输入 hugo server。如果出现：

或类似的界面，就说明主题安装成功了。你可以在浏览器里输入 http://localhost:1313 来欣赏一下自己的新网站。到这一步，大家就有了一个网站啦！

大家可以自行尝试修改 [example] 下的 config.toml 文件，一定程度上改变网站设置。

注：网上更流行的一种方法是使用 git 安装，使用该方法需要电脑里预装 git。使用命令：

1
2
3

git init
git submodule add https://github.com/heyeshuang/hugo-theme-tokiwa.git

配置方法与上述类似。

5） 新建文章

打开命令行窗口，在D:\hugo\sites\[example] 下输入 hugo new post\[YourPost].md，YourPost 替换成你的文章名字。 你会发现D:\hugo\sites\[example]\contents\posts文件夹下出现了一个 Markdown 文档。文档开头有一份类似这样的文字：

1
2
3

title: "用hugo搭建博客 1"
date: 2020-08-03T14:18:38+08:00
draft: true

正常使用 Markdown 语言编写文档即可。Markdown 语法简介

文档编写完毕后，需要将 draft: true 改为 draft: false，文章才能正常发布。注意，所有标点都必须是英文标点，冒号后面要有空格，否则机器编译时会出错。

hugo new about.md 会在 contents 列表下直接产生一个 about.md 文件，填写后会出现在网站的 about 一栏里。这里有更多可以投喂给你的 hugo 的指令。

好了，到此为止，我们已经有了一个能在本地运行的 hugo 博客了。

6） 上传到 GitHub

注意：

  • hugo 不是非得托管在 GitHub 上的。至少有 GitLab 和 BitBucket 也能提供同样的托管服务，但是！官网文档上对 GitHub 托管的前置要求是：

    1. You have Git 2.8 or greater installed on your machine.
    2. You have a GitHub account. Signing up for GitHub is free.
    3. You have a ready-to-publish Hugo website or have at least completed the Quick Start.

    而对 GitLab 和 Bitbucket 的前置要求里都有这么一句：

    1. Working familiarity with Git for version control

    像我这样菜的人……看到 familiarity 直接绕道走吧。所以，嗯，对不起程序员们。

  • 你可以下一个 Git 然后按照网上的教程或者官网指导做，这些教程很多的，如果你沿着上传博客的教程做不通你还可以搜一下“程序员怎么把程序上传给 github”。但是，如果你是个小白的话，这件事会变得非常非常困难。你有很大概率会发现即使你每一步都和教程打得一样，你还是会有文件找不到 / 程序给你大量报错。所以如果没有特殊的理想与爱好的话，建议下一个 GitHub 桌面端。MacOs 和 Windows 系统都有桌面端应用程序，可以让你以小白能够驾驭的方式上载文件。Linux 是没有的，但话说回来……用 Linux 的大佬为什么要看我一个连 Git 都用不会的菜鸡的文章啊？！

所以，我下了一个 Github 桌面端。如果嫌官网下载慢的话可以点这里，提取码 kiph，虽然我觉得如果你不充钱的话下载速度比官网快不了多少。然后事情就变得简单了。

首先打开命令行界面，在命令行界面里进入你的网站文件夹 D:\hugo\sites\[example] ，输入指令 hugo。系统会在 D:\hugo\sites\[example] 下生成一个文件夹 public。你只需要将这个文件夹上传到云端，他人即可看到你的网站。

去 github.com 注册一个账号，回来在桌面端上登录。然后选择“从本地新建库”，选择你的 public 文件夹。这个库的名字一定要是 [你的用户名].github.io。根据桌面端的指示，在上载（commit）完成后即可 push 到云端。因为这一段有 GUI 的帮助，所以操作会变得非常简单易懂。当桌面端提示你上载完成后，你就可以尝试着在浏览器里输入 [你的用户名].github.io 来进入你的网站。

当你在本地修改了博客文件并已经通过 hugo server 指令确定网站能正常运转后，再次运行 hugo 指令，然后去桌面端点 commit 再点 push，网站就可以更新了。

你也可以通过创建库然后绑定其他网站如 netlify.com 的方式创建网页。netlify 有一个小小的优势就是它内置了 CDN，速度可能会快一些。具体方式真的没必要写了，网站的说明非常非常详细且清晰易懂。

7） 设置个性域名

这一步我没有做。我还没有经济独立，对我来说一个 subdomain 已经够了，我不想花家长的钱买一个时髦酷炫的 .com，也不是很想注册一个 .tk 或者 .ml。所以我目前就挂在一个 subdomain 上，如果你能看到这篇文章，你应该就能看到我的地址。

如果你想要的话，可以买一个域名。据说国内买的话要实名制登记，国外的话不需要。然后绑定 GitHub Pages 的教程 这里 有，注意我没跟着做过，也不知道能不能行。Netlify 直接有绑定个性域名这一步，跟着指示做就好。

嗯，就这样。如果有人看了这个后做了个 Hugo，我感激不尽。你别想了不会有人折腾这玩意的。
